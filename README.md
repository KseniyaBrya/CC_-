# SQL Analytics Agent

CLI-агент, который принимает вопрос на естественном языке, генерирует SQL с помощью **xAI Grok**, выполняет запрос к PostgreSQL и возвращает результат в текстовом виде.

---

## Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone https://github.com/your-username/sql_agent.git
cd sql_agent

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Создать файл с переменными окружения
cp .env.example .env
# Открыть .env и вписать XAI_API_KEY и параметры PostgreSQL

# 4. Создать таблицы и загрузить тестовые данные
python setup_db.py

# 5. Запустить агента
python agent.py                                     # интерактивный режим
python agent.py -q "Топ-5 регионов по выручке"    # разовый запрос
```

---

## Где получить XAI_API_KEY

1. Перейти на **https://console.x.ai**
2. Зарегистрироваться / войти
3. Раздел **API Keys** → **Create Key**
4. Скопировать ключ вида `xai-...` в файл `.env`

## Архитектура пайплайна

```
Пользователь (CLI)
      ↓  вопрос на русском
sql_generator.py
      ↓  системный промпт + схема БД + вопрос → xAI Grok API
      ↓  JSON-ответ: { sql, explanation, is_safe }
      ↓  проверка безопасности (только SELECT, без DDL/DML)
db.py
      ↓  выполнение SELECT в read-only транзакции
      ↓  QueryResult { columns, rows, error }
formatter.py
      ↓  адаптивное форматирование: одно значение / список / таблица
Пользователь (CLI)
```

### Как генерируется SQL

Модуль `sql_generator.py` отправляет в Grok:
- **Системный промпт** со схемой БД и правилами безопасности
- **Вопрос пользователя**

Grok возвращает JSON `{ sql, explanation, is_safe }`.
После этого `sql_generator.py` дополнительно валидирует SQL:
- запрос должен начинаться с `SELECT`
- запрос не должен содержать `INSERT / UPDATE / DELETE / DROP / ALTER` и др.

### Ограничения и допущения

- Работает только с таблицами `orders` и `customers`
- Только SELECT-запросы (read-only транзакция на уровне PostgreSQL)
- Одиночный запрос — без диалогового контекста
- При неоднозначном вопросе агент выбирает наиболее разумную интерпретацию

## Схема базы данных

### `orders`
| Колонка       | Тип     | Описание              |
|---------------|---------|-----------------------|
| `order_id`    | INTEGER | ID заказа             |
| `order_date`  | DATE    | Дата заказа           |
| `customer_id` | INTEGER | ID покупателя         |
| `region`      | TEXT    | Регион                |
| `city`        | TEXT    | Город                 |
| `amount`      | NUMERIC | Сумма заказа          |
| `status`      | TEXT    | completed / cancelled |

### `customers`
| Колонка       | Тип     | Описание                         |
|---------------|---------|----------------------------------|
| `customer_id` | INTEGER | ID покупателя                    |
| `gender`      | TEXT    | Пол (M / F)                      |
| `age`         | INTEGER | Возраст                          |
| `segment`     | TEXT    | Сегмент (premium / standard / …) |
